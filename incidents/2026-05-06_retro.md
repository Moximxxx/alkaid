## 根因分析

### 问题：Stream参数设置但API返回非流式响应

**现象**：
- 请求已正确设置 `stream: true`
- 但 minimax API 返回的是非流式完整响应（使用 `choices[0].text` 而非 `choices[0].delta.content`）

**根因**：
minimax API 的流式响应格式与标准 OpenAI 兼容格式存在差异：
- 标准 OpenAI 兼容格式：`choices[0].delta.content`
- minimax 非流式格式：`choices[0].text`
- minimax 流式格式：同样使用 `choices[0].text`

**修复方案**：
同时支持 `delta.content`、`text`、`message.content` 三个字段，取第一个有值的字段。

## 经验教训

### 1. API 兼容层设计的重要性

**问题**：不同 AI provider 的响应格式存在差异。

**教训**：
- 不能假设 OpenAI 兼容 API 的响应格式完全一致
- 需要针对每个字段设计fallback机制
- 建议维护每个 provider 的格式差异文档

### 2. 调试日志的必要性

**改进**：添加了完整的请求/响应日志：
- `[AI] sendMessage called`
- `[AI] API request body:`
- `[AI] Chunk received:`
- `[AI] Parsed content:`
- `[AI] onFirstToken called`
- `[AI] onComplete called`

**价值**：便于快速定位是请求问题还是响应处理问题。

### 3. 错误处理不能静默吞掉

**改进**：从 `catch {}` 改为：
```typescript
catch (e) {
  console.error('流式解析错误:', e, '原始数据:', data)
  console.log('[AI] Error:', e)
}
```

**价值**：保留错误信息便于后续排查。
