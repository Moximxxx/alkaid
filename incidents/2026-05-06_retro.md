## 问题分析与修复

### 问题描述

**问题**：代码使用硬编码的 API 配置，无法支持用户自定义 API Key

**根因分析**：
- `ai.ts` 中 `MODEL_CONFIG` 硬编码了 provider、model、apiKey
- `Home.tsx` 使用 `setTimeout` mock 回复，不是真实 AI 调用

### 修复方案

| 文件 | 修改内容 |
|------|---------|
| ai.ts | `useAI` 改为接收 `UseAIOptions` 对象，支持外部传入 `apiKey` |
| Home.tsx | 移除 mock，通过 `useSettings` 获取配置，调用真实 `sendMessage` |

## 经验教训

### 1. 配置外部化原则

**问题现象**：
- 硬编码配置导致无法灵活切换 provider 和 model
- API Key 写死代码中不安全

**解决方案**：
- 配置通过 props/context 注入
- 支持外部传入 apiKey 覆盖默认

**关键点**：
```
❌ 错误：const apiKey = import.meta.env.VITE_XXX
✅ 正确：const apiKey = externalApiKey || import.meta.env.VITE_XXX
```

### 2. Mock 代码移除时机

**问题现象**：
- Mock 代码可能长期遗留，导致功能看似正常实则虚假

**经验**：
- Mock 代码应有明确的移除计划
- 建议在 TODO 注释中标注 "MOCK:" 前缀，便于识别和清理

### 3. 错误处理的重要性

**本次改进**：
- 添加 `try-catch` 捕获 sendMessage 异常
- 用户可见错误信息，而非静默失败
- loading 状态防止重复提交
