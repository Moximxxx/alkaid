## 经验教训

### 1. 流式 UI 更新机制

**本次改进**：
- 使用 `messageUpdateCallback` 回调实现增量更新
- UI 组件通过 `setMessages` 合并更新（存在则替换，不存在则添加）

**关键点**：
```typescript
// 增量更新模式
setMessages((prev) => {
  const exists = prev.some((m) => m.id === msg.id)
  if (exists) {
    return prev.map((m) => (m.id === msg.id ? { ...m, content: msg.content } : m))
  }
  return [...prev, { id: msg.id, role: msg.role, content: msg.content }]
})
```

### 2. SSE 流式数据解析

**技术细节**：
- 响应格式：`data: {"choices":[{"delta":{"content":"xxx"}}]}\n\n`
- 结束标志：`data: [DONE]`
- 需要处理 TCP 粘包：buffer 暂存，split('\n') 分割

**关键点**：
```typescript
buffer += decoder.decode(value, { stream: true })
const lines = buffer.split('\n')
buffer = lines.pop() || ''  // 保留未完整的一行
```

### 3. loading 状态的 UI 联动

**最佳实践**：
- 发送时：清空输入框 → 设置 loading
- 收到流式数据：保持 loading
- 接收完毕：loading 变为 false
- 输入框 placeholder 根据 loading 状态动态变化
