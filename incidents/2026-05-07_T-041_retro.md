## 经验教训

### 1. 代码变更必须立即提交

**问题**：T-039 完成了代码修复，但未提交到 git

**教训**：
- 代码修复完成后**立即提交**，不要停留在工作目录
- 提交是工作流闭环的必要条件
- 建议在任务完成检查清单中明确标注"已提交"

### 2. 构建验证必须在提交后执行

**问题**：T-040 声称构建验证通过，但代码未提交

**教训**：
- Builder 必须在代码提交后才能执行构建验证
- 未提交的代码无法保证构建产物的完整性
- 建议 Builder 在构建前检查 `git status`，如有未提交变更则拒绝构建

### 3. Coordinator 必须验证闭环状态

**问题**：Coordinator 委派了 T-039 和 T-040，但未检测到工作流未闭环

**教训**：
- Coordinator 在 Builder 验证后应确认代码已提交
- 复盘前必须验证所有任务已完成闭环
- 建议在 Retro 之前检查 git log 确认提交记录
