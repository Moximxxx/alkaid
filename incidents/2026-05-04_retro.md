## 问题分析与修复

### 问题描述

**问题**：Routes 嵌套导致 HomePage 无法获取正确高度，输入框无法粘底

**根因分析**：
- App.tsx 的 `<main>` 区域未设置正确的 flex 布局
- HomePage 的输入区域使用 `fixed` 定位与 Toolbar 重合
- Routes 嵌套后 HomePage 无法正确继承父容器高度

### 修复方案

| 文件 | 修改内容 |
|------|---------|
| App.tsx | main 添加 `flex-1 flex flex-col overflow-hidden` |
| Home.tsx | 根容器添加 `relative`，输入区域改用 `absolute bottom-0` |

### 修复后布局结构

```
App.tsx (min-h-screen flex flex-col)
├── Toolbar (固定高度)
├── Sidebar (overlay)
└── main (flex-1 flex flex-col overflow-hidden relative)
    └── HomePage (h-full flex flex-col relative)
        ├── Chat Area (flex-1 overflow-hidden)
        │   └── h-full overflow-y-auto
        └── Input Area (absolute bottom-0 left-0 right-0)
```

## 经验教训

### 1. Routes 嵌套下的 Flex 布局高度传递

**问题现象**：
- Routes 嵌套时，子路由组件（如 HomePage）默认无法获取父容器（main）的剩余空间高度

**解决方案**：
- 父容器（main）必须设置 `flex-1 flex flex-col overflow-hidden` 建立明确的 flex 上下文
- 子组件使用 `h-full` 继承父容器高度

**关键点**：
```
父容器 flex 布局 → 明确高度 → 子组件 h-full 才能生效
```

### 2. 粘底布局的实现方式对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| `fixed bottom-0` | 简单直接 | 脱离文档流，可能与 Toolbar/Header 重合 |
| `absolute bottom-0` | 相对于最近定位祖先 | 需要祖先有 `relative` |
| `flex-shrink-0` | 参与 flex 布局 | 需要整体滚动区域配合 |

**本次选择**：`absolute bottom-0` + `relative` 父容器
- 原因：输入区域需要独立于 Chat Area 滚动，不被 Chat Area 的 overflow 影响

### 3. 输入区域设计原则

- 输入框应该固定在视口底部，不随聊天内容滚动
- 使用 `backdrop-blur` 实现毛玻璃效果，提升视觉层次
- 配合 `border-t` 明确分隔边界
