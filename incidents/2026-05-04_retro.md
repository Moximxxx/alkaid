## 经验教训

1. **UI 状态反馈的重要性**：保存操作后用户需要明确的成功提示，这是基本用户体验要求

2. **布局高度选择**：
   - `h-screen` vs `h-full`：在 flex 布局中，容器高度应使用 `h-screen` 确保占满视口
   - `flex-shrink-0`：输入区域不应被压缩，确保内容不被挤压变形

3. **设置完成标志**：使用 `setupCompleted` 而非仅依赖 API Key 判断配置状态，更准确反映用户意图

4. **工作流闭环价值**：即使是小任务，也要完整执行 R-6 闭环，确保质量
