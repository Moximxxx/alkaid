## verify_arch.sh P-01 误报分析

### 误报现象
```
[ERROR] P-01: AI 模型列表不一致:  doubao (Welcome:5 vs constants:8)  openai (Welcome:3 vs constants:4)
```

### 实际情况
**Welcome.tsx** (5 个 doubao 模型):
- Line 17: doubao-2.0-vision-pro
- Line 18: doubao-1.5-thinking-vision-pro
- Line 33: doubao-seed-2.0-pro
- Line 34: doubao-2.0-pro
- Line 35: doubao-2.0-lite

**constants.ts** (8 个 doubao 引用，但实际模型只有 5 个):
- Line 16: `model: 'doubao-2.0-vision-pro',` ← **误将 DEFAULT_AI_CONFIG.model 计入**
- Line 38-39: VISION_MODELS.doubao_vision (2个)
- Line 42-44: TEXT_MODELS.doubao_text (3个)
- Line 99-100: VISION_MODELS 数组重复 (2个) ← **误将同一数组的多次定义计入**

### 根因
verify_arch.sh 使用 `grep -c 'doubao-'` 全局搜索：
1. 会将配置对象中的默认值（如 DEFAULT_AI_CONFIG.model）误计入
2. 无法识别同一数组在不同位置的重复定义
3. 缺乏对 MODELS 对象结构的语义理解
