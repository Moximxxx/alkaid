## 经验教训

### 1. 嵌套 Flex 布局的 Height 传递机制

**问题**：为什么 `h-full` 能正确填充 main 的高度？

**原理**：
- `flex-1` 使 main 占据父容器（App）的剩余空间
- 剩余空间由 App 的 `min-h-screen flex flex-col` 分配
- Home 使用 `h-full` 表示"100% 父容器高度"
- 父容器（main）有明确高度（由 flex-1 计算得出），所以 `h-full` 生效

**关键点**：
```
父容器必须有明确高度 → h-full 才能生效
```

### 2. Chat Area 溢出处理

| 样式 | 效果 |
|------|------|
| `overflow-hidden` | 容器本身不显示溢出内容 |
| `overflow-y-auto` | 内容区内部可滚动 |

**当前设计**：
- Home → Chat Area (`flex-1 overflow-hidden`)：占满剩余空间
- Chat Area → 内容 wrapper (`h-full overflow-y-auto`)：内容滚动

### 3. Input Area 固定策略

**错误方案**：
- `position: fixed bottom-0`：脱离文档流，可能与 Toolbar 冲突

**正确方案**：
- `flex-shrink-0`：在 flex 布局中保持最小高度
- `border-t`：视觉上分隔
