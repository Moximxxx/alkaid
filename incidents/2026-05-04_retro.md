## 问题分析

### 当前布局结构（Home.tsx 第63行）

```jsx
<div className="flex flex-col h-screen fixed inset-0">
```

**问题**：
- `fixed inset-0`：将容器固定在视口，覆盖整个窗口
- `h-screen`：高度固定为视口高度
- 聊天区域 `overflow-hidden` + 内部 `overflow-y-auto`

**现象**：
- 整个页面被固定，内容超出时只有聊天区域内部滚动
- 输入框始终可见，但主页整体无法滚动

### 布局层级

```
fixed inset-0 h-screen (第63行)
├── Chat Area (flex-1 overflow-hidden)
│   └── overflow-y-auto (第66行)
└── Input Area (flex-shrink-0)
```

## 经验教训

### 1. 固定布局 vs 滚动布局

**当前问题布局**：
```jsx
<div className="fixed inset-0 h-screen">
```
- `fixed`：脱离文档流，固定于视口
- 内容超出时无法触发外层滚动

**正确滚动布局**：
```jsx
<div className="min-h-screen">  // 或 h-screen + overflow-y-auto
```

### 2. Flex 布局中的滚动控制

| 需求 | 外层容器 | 内容区 | 输入区 |
|------|---------|--------|--------|
| 整体滚动 | `min-h-screen` | `flex-1 overflow-y-auto` | `flex-shrink-0` |
| 固定底部输入 | `h-screen flex flex-col` | `flex-1 overflow-hidden` | `flex-shrink-0` |

### 3. 验证脚本报告的历史遗留问题

- verify_arch.sh 报告的 P-01 错误是历史遗留问题
- 不影响本次 UI 修复任务的验收
- 需要单独委派任务修复 AI 模型列表一致性
