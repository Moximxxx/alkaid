# INC-2026-0510-001: 进程清理误杀 OpenCode 主进程

## 基本信息

- **日期**: 2026-05-10
- **事故编号**: INC-2026-0510-001
- **报告人**: Coordinator
- **严重级别**: 中（导致工具执行中断，需手动重启 OpenCode）

## 事故描述

在执行 E2E 测试后的进程清理环节，使用以下命令无差别杀进程：

```powershell
Get-Process -Name "node","bun" -ErrorAction SilentlyContinue | Stop-Process -Force -ErrorAction SilentlyContinue
```

该命令将 OpenCode 主进程（也可能运行在 Node.js 或 bun 上）一并杀死，导致工具执行中断。

## 根因分析

1. **无差别匹配进程名**：`-Name "node","bun"` 匹配所有同名进程，OpenCode 主程序也使用相同的运行时
2. **未追踪子进程 PID**：启动 Vite dev server 时未记录具体 PID，导致清理时无法精确定位
3. **缺少守护机制**：没有使用进程树管理或作业对象来隔离测试进程

## 影响范围

- OpenCode 主程序中断
- 当前对话工具执行失败
- 需要手动重启 OpenCode

## 修复措施

### 短期修复（立即）
1. 创建 smoke-tester 智能体时内置安全进程管理规则
2. 启动后台进程时追踪具体 PID，而非匹配进程名
3. 进程清理使用 `Stop-Process -Id $specificPid` 而非 `-Name`

### 长期改进
1. 在 smoke-tester 的 prompt 中明确禁止无差别杀进程
2. 使用 `Start-Job` + `$job.Id` 追踪子作业
3. 端口检测 + PID 文件方式管理进程生命周期
4. 在 builder 和 retro 的约束中也增加此规则

## 安全进程管理规范（新增）

```
启动后台进程时:
  $process = Start-Process -NoNewWindow -PassThru -FilePath "bun" -ArgumentList "run dev"
  $processId = $process.Id
  # 将 $processId 写入临时文件以便后续精确清理

清理后台进程时:
  $pid = Get-Content ".opencode/tmp/vite.pid" -ErrorAction SilentlyContinue
  if ($pid) { Stop-Process -Id $pid -Force -ErrorAction SilentlyContinue }
  # 绝不使用 Stop-Process -Name "node","bun"
```
