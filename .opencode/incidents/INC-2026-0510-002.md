# INC-2026-0510-002: 后台服务启动阻塞导致子 Agent 卡死

## 基本信息
- **日期**: 2026-05-10
- **事故编号**: INC-2026-0510-002
- **严重级别**: 中

## 事故描述
smoke-tester 子 Agent 在执行 `Start-Job { bun run dev }` 启动 Vite 开发服务器时卡死。后台进程管理不当导致 Agent 执行超时无响应。

## 根因分析
1. **子 Agent 不适合管理长任务**：子 Agent 的 bash 工具有超时限制，启动 Vite 这类常驻进程会导致 Agent 阻塞
2. **缺少后台服务专用 Agent**：没有隔离机制让后台服务与测试 Agent 分离
3. **缺少心跳监控**：没有定期检查服务状态的机制

## 解决方案
引入两个新 Agent 角色：

### 后台服务 Agent (service-agent)
- 职责：启动/停止常驻后台服务（Vite、Electron 等）
- 启动方式：`Start-Process -WindowStyle Hidden` 完全分离进程
- 返回：PID（写入 `.opencode/tmp/{service}.pid`）
- 停止方式：按 PID 精确杀进程

### 心跳 Agent (heartbeat)
- 职责：监控指定服务是否存活并就绪
- 检查项：PID 是否存在 + 端口是否可达
- 返回状态：`READY` / `WAITING` / `DEAD`
- 每次调用独立，Coordinator 循环调用直到 READY 或超时

## 新增规则
- 子 Agent 禁止启动常驻后台进程
- 后台进程必须由 service-agent 启动
- 后台服务启动后必须有心跳 Agent 验证就绪
