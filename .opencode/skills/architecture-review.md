# Architecture Review Skill

## 使用场景

当用户需要进行架构评审、审查技术方案、或评估系统设计时使用此 Skill。

## 评审维度

### 1. 架构原则

| 原则 | 说明 |
|------|------|
| 单一职责 | 每个模块只做一件事 |
| 开闭原则 | 对扩展开放，对修改关闭 |
| 里氏替换 | 子类可以替换父类 |
| 接口隔离 | 多个专用接口优于一个通用接口 |
| 依赖反转 | 依赖抽象而非具体 |

### 2. 质量属性

| 属性 | 关注点 |
|------|--------|
| 性能 | 响应时间、吞吐量 |
| 可用性 | uptime、降级 |
| 可扩展性 | 水平扩展、垂直扩展 |
| 可维护性 | 代码复杂度、耦合度 |
| 安全性 | 认证、授权、数据保护 |

## 三层架构验证

### R-01: 分层职责明确

```
┌─────────────────────────────────┐
│         Layer 3: UI              │  ← 前端/展示层
│   (React, Vue, Android, iOS)   │
└─────────────────────────────────┘
                  ↓
┌─────────────────────────────────┐
│      Layer 2: Business Logic    │  ← 业务逻辑层
│      (Services, Use Cases)      │
└─────────────────────────────────┘
                  ↓
┌─────────────────────────────────┐
│      Layer 1: Data Access      │  ← 数据访问层
│   (Repositories, DAOs, APIs)   │
└─────────────────────────────────┘
```

### R-02: 禁止跨层调用

```bash
# 错误：UI 直接调用数据访问层
UI → Repository

# 错误：业务逻辑直接调用 UI
Service → UI

# 正确：单向依赖
UI → Service → Repository
```

### R-03: 跨层通信必须通过接口

```python
# 正确：依赖抽象
class UserService:
    def __init__(self, user_repo: IUserRepository):
        self.user_repo = user_repo

# 接口定义
class IUserRepository(ABC):
    @abstractmethod
    def get_user(self, user_id: str) -> User: pass
```

## 模块检查

### R-04: 模块内聚性

| 内聚类型 | 说明 | 推荐 |
|----------|------|------|
| 功能内聚 | 模块只做一件事 | ✅ |
| 顺序内聚 | 按顺序执行相关任务 | ⚠️ |
| 通信内聚 | 操作相同数据 | ⚠️ |
| 过程内聚 | 按特定顺序执行 | ❌ |
| 巧合内聚 | 无关代码放一起 | ❌ |

### R-05: 模块耦合度

| 耦合类型 | 说明 | 推荐 |
|----------|------|------|
| 无耦合 | 完全独立 | ✅ |
| 数据耦合 | 通过参数传递 | ✅ |
| 印记耦合 | 传递不必要数据 | ⚠️ |
| 控制耦合 | 传递控制标志 | ❌ |
| 内容耦合 | 直接访问内部 | ❌ |

## 设计模式检查

### R-06: 过度工程的标志

```python
# 错误：过早优化和不必要的复杂性
class UserManagerSingletonFactoryBuilder:
    _instance = None

    @classmethod
    def get_instance(cls):
        if cls._instance is None:
            cls._instance = cls.__new__()
        return cls._instance

# 正确：简单直接
class UserService:
    pass
```

### R-07: 常用设计模式场景

| 场景 | 推荐模式 |
|------|---------|
| 依赖注入 | Constructor Injection |
| 策略切换 | Strategy |
| 单例（必要时） | Singleton |
| 工厂创建 | Factory Method |
| 观察者事件 | Observer |
| 缓存 | Cache-Aside |
